# 2026-06-14 GitHub Issue Drafts

GitHub App 권한 문제로 이슈 자동 생성이 실패해서, 아래 내용을 복사해 이슈로 등록한다.

## 1. [P0] 첫 소셜 로그인 세션 확인 실패 수정

### 문제
신규 사용자가 Google/Kakao 소셜 로그인을 처음 시도할 때 한 번은 거부되고, 다음 문구가 노출된다.

> 소셜 로그인 세션 확인에 실패했습니다. 같은 브라우저에서 다시 로그인해주세요.

### 확인할 것
- Supabase OAuth callback 후 session exchange 흐름
- PKCE verifier 저장/복구 타이밍
- redirect URL, origin, TWA/브라우저별 callback 차이
- 첫 로그인과 재로그인의 session 상태 차이

### 완료 기준
- 신규 Google/Kakao 로그인 첫 시도에서 정상 진입
- 기존 계정 재로그인도 정상 동작
- 실패 시 사용자가 이해할 수 있는 안내 표시

## 2. [P0] 식단표 이미지 저장 시 잘림 수정

### 문제
식단표를 이미지로 저장하면 일부 영역이 잘려 저장된다.

### 확인할 것
- 캡처 대상 DOM 크기와 실제 스크롤 영역 차이
- 모바일 viewport, safe-area, fixed bottom nav 영향
- html-to-image/canvas 변환 시 scale, width, height 설정

### 완료 기준
- 모바일에서 전체 식단표가 잘리지 않고 저장됨
- 긴 식단표도 누락 없이 저장됨
- 저장 전/후 이미지 비율이 자연스러움

## 3. [P0] 영수증 스캔 UX 개선

### 문제
냉장고 재료 추가에서 영수증 스캔 흐름이 사용자가 이해하기 어렵다.

### 개선 방향
- 스캔 버튼 문구와 상태를 명확히 표시
- 이미지 선택/촬영 후 처리 중 상태 표시
- 실패 시 재시도 가능하게 안내
- 인식 결과를 바로 저장하지 않고 사용자가 확인 후 추가

### 완료 기준
- 사용자가 촬영, 분석, 확인, 추가 흐름을 자연스럽게 이해함
- 실패/빈 결과/부분 인식 상태가 모두 처리됨

## 4. [P1] 주요 페이지 상단 sticky 처리

### 요구사항
- 냉장고 페이지: 재료 검색창과 카테고리 필터까지 상단 고정
- 레시피북 페이지: 검색 영역과 탭 네비게이션까지 상단 고정
- 마이페이지: `<- 마이페이지` 헤더 고정

### 완료 기준
- 스크롤 시 주요 조작 영역이 사라지지 않음
- bottom nav, safe-area와 겹치지 않음
- iOS/Android 모바일 브라우저에서 레이아웃 깨짐 없음

## 5. [P1] 홈 화면에 이유식 준비 가이드 추가

### 요구사항
홈 화면의 아기 카드 아래에 `이유식 준비 가이드` 섹션을 추가한다.

### 디자인 기준
- 기존 냠픽 디자인 가이드 유지
- 한국어 문장 줄바꿈이 어색하지 않게 배치
- 글자 크기, 색상, 간격을 기존 홈 화면과 맞춤
- 긴 문장은 의미 단위로 자연스럽게 다음 줄로 넘김

### 완료 기준
- 가이드 카드가 아기 카드 아래에 자연스럽게 배치됨
- 누르면 기존 HTML 프로토타입과 같은 내용/동작 제공
- 모바일에서 문장 마지막 줄이 1~2글자만 남지 않음

## 6. [P1] 식단표 이미지/차트 구성 재설계

### 문제
현재 식단표 이미지 구성이 보기 좋지 않고 저장 이미지도 개선 필요가 있다.

### 참고
- Notion에 있는 식단표 레퍼런스 확인 후 반영

### 완료 기준
- 저장용 식단표 이미지가 공유하기 좋은 구성으로 정리됨
- 날짜, 아기 정보, 끼니별 메뉴가 명확히 보임
- 모바일 저장 이미지에서 잘림 없음

## 7. [P1] 로그인/회원가입 화면 개선 및 약관 연결

### 요구사항
- 로그인 화면 디자인 개선

- 마이페이지 약관 화면과 연결

- 로그인/회원가입 UI가 냠픽 스타일과 일관됨

## 8. [P2] 마이페이지 준비중 기능 정리

### 요구사항
- 알레르기 관리 페이지 준비중 처리
- 비밀번호 변경 준비중 처리

### 완료 기준

- 추후 구현할 기능 범위가 문서화됨

## 9. [P2] 아기 정보 수정 기능 추가

### 문제
현재 아기 정보를 등록한 뒤 수정하는 기능이 없다.

### 완료 기준
- 아기 이름/생년월일 등 주요 정보 수정 가능
- 수정 후 홈 화면과 식단 화면에 즉시 반영
- 잘못된 입력값 검증

## 10. [P2] 랜딩/아이콘/유입 경로 조사 정리

### 요구사항
- 랜딩페이지 이미지를 손그림 스타일로 정리
- 보호자 이름, 아기 이름 예시를 별명 기반으로 자연스럽게 표시
- 메인/앱 아이콘을 최종 아이콘으로 통일
- 가입 또는 온보딩 중 `어느 경로로 이 앱을 알게됐나요?` 질문 추가 검토

### 완료 기준
- public icon, manifest, TWA icon, 헤더 아이콘이 일관됨
- 랜딩 이미지가 배경 문제 없이 표시됨
- 유입 경로 응답 저장 방식 결정

## 11. [P1] 앱 재진입 시 인증 화면 노출 최소화

### 문제
앱을 나갔다가 다시 들어왔을 때 로그인 유지 상태라면 굳이 로그인 화면이 다시 보일 필요가 없다.

### 확인할 것
- Supabase session 복구 시점
- `AuthLoadingView`가 사용자에게 너무 길게 노출되는지
- `nyampick:has-session` 캐시와 실제 session 상태 차이
- TWA/모바일 브라우저 재진입 시 초기 라우팅 흐름

### 완료 기준
- 로그인 유지 상태에서는 auth 화면이 불필요하게 노출되지 않음
- 세션 확인 중에는 짧고 일관된 로딩 화면만 보임
- 세션 만료 시에만 로그인 화면으로 이동

## 12. [P1] 공통 스크롤바 노출 제거

### 문제
일부 화면 옆에 스크롤바가 보여 모바일 앱처럼 보이지 않는다.

### 확인할 것
- body/html/app shell overflow 설정
- 페이지별 가로 스크롤 발생 원인
- sticky header, bottom nav, safe-area와의 충돌 여부

### 완료 기준
- 모바일 주요 화면에서 시각적 스크롤바가 보이지 않음
- 실제 스크롤 동작은 유지됨
- 가로 스크롤이 발생하지 않음

## 13. [P2] 회원탈퇴 준비중 처리

### 문제
회원탈퇴 기능이 아직 완성되지 않았다면 빈 화면이나 오류 대신 명확한 준비중 안내가 필요하다.

### 완료 기준
- 회원탈퇴 진입 버튼 또는 영역 클릭 시 준비중 안내 표시
- 실제 탈퇴 구현 전까지 사용자가 혼동하지 않음
- 추후 탈퇴 구현 범위가 문서화됨

## 14. [P2] 가족구성원 역할명 변경 검토

### 요구사항
현재 가족구성원 테마의 역할명이 `배우자`, `할머니`, `할아버지`, `가족구성원` 중심이라면 더 넓게 쓸 수 있도록 변경을 검토한다.

### 변경 후보
- 배우자
- 가족
- 친구
- 도우미

### 완료 기준
- 초대/가족 구성원 화면의 역할명이 더 자연스럽게 정리됨
- 기존 데이터와 충돌하지 않음
- 화면 문구와 저장 값의 관계가 명확함
